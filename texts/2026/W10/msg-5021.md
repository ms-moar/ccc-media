OpenAI Symphony                                                                                                      
                                                                                                                             
  Что это                                                                                                                    
                                                                                                                             
  Оркестратор для автономных кодинг-агентов. Symphony — демон, который:                                                      
  1. Поллит трекер задач (сейчас — Linear) каждые 30с                                                                        
  2. Для каждой задачи создаёт изолированный workspace (git clone)                                                           
  3. Запускает кодинг-агента (Codex app-server) в этом workspace                                                             
  4. Агент работает автономно: пишет код, создаёт PR, ждёт ревью        
  5. Менеджерит lifecycle: retry с backoff, concurrency limits, stall detection, terminal cleanup                            

  Ключевая идея: инженеры управляют работой (Linear board), а не агентом. Задачу двигаешь в "In Progress" → Symphony
  подхватывает и делает.

  Юзкейс

  - Командная автоматизация рутинных тасков (баг-фиксы, типовые фичи, рефакторинг)
  - "Harness engineering" — кодбейз с хорошими тестами, CI, CLAUDE.md/codex setup → агенты работают автономно
  - Масштабирование: до 10 параллельных агентов, per-state лимиты, multi-turn сессии

Интеграция с Claude Code

  Прямая интеграция невозможна — Symphony заточен под Codex app-server protocol (line-delimited JSON на stdout, специфичный
  handshake: initialize → thread/start → turn/start).
