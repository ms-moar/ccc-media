💥 **CloudFlare запустила убийцу WordPress - EmDash** 💥
Если вы хоть раз обновляли WordPress-плагин и молились, чтобы сайт не упал — теперь на вашей улице праздник! 😄

**Итак, кратно что за зверь:**
👉 TypeScript вместо PHP — современный стек, на котором пишут современные разрабы и агенты
👉 Теперь все плагины запускаются в отдельных «песочницах» и фактически общаются с основным ядром системы по API - это огромный буст в плане безопасности т.к. 96% всех уязвимостей сайтов на WordPress приходится на плагины
👉 Сайты разворачиваются по механике Edge - «на лету» и из кучи разных мест мира. Это как CDN на максималках. И в целом это Server-less механика, что очень круто! (гуглим)
👉 Есть возможность хранить только статичные отрендеренные HTML-страницы в CloudFlare Pages! А это значит что цена владения небольшим сайтом (до 100к визитов в день) = цена продления доменов. И да - никаких лагов сайтов из-за Shared-хостингов. Да - защищенная песочница для плагинов обойдется в $5 в месяц. Зато не хакнуть.
👉 Теперь темы на Astro вместо PHP-шаблонов из 2005-го. Astro - фреймворк для контентных сайтов, рендеринга тем

**Максимальная интеграция с ИИ - фактически это ИИ-Native CMS: MCP, CLI, Agent Skills**
👉 Встроенный MCP-сервер — Claude, ChatGPT и другие ИИ-инструменты могут управлять сайтом напрямую через Model Context Protocol - просто пишете Claude __«Замени в Title и контенте всех страниц год с 2025 на 2026»__ и он заменит.
👉 Agent Skills — файлы-инструкции для ИИ-агентов: как писать плагины, портировать темы с WordPress, работать со схемами контента
👉 👉 Дать своему агенту прямое управление сайтом через эту штуку [https://github.com/emdash-cms/emdash/blob/main/docs/src/content/docs/reference/cli.mdx](https://github.com/emdash-cms/emdash/blob/main/docs/src/content/docs/reference/cli.mdx)
👉 CLI — программное управление контентом, медиа, шаблонами страниц - все легко интегрируется с агентами


**Легкий переезд с WordPress**
В EmDash можно быстро импортировать уже существующие WordPress-сайты.
**Есть несколько вариантов:**
👉 Экспорт WXR-файла (WordPress eXtended RSS) из WordPress-админки
👉 Через плагин EmDash Exporter, который создаёт защищённый endpoint для миграции - своего рода визард где тыкаешь что перенести и он переносит
👉 Автоматический перенос постов, страниц, медиафайлов и таксономий
👉 Кастомные типы контента тоже можно перенести через визуальный конструктор схем в админке. Для тех у кого много хардкора на сайтах 😁
