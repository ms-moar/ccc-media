**🤖 Claude Code отупішав: що сталось і як пофіксити**

Директор AI в AMD проаналізувала 6 852 сесії Claude Code і виклала все на GitHub. 234 760 tool calls. 17 871 thinking blocks. Висновок: "Claude cannot be trusted."

Що сталось? Anthropic тихенько зробили дві речі:
- Включили adaptive thinking - де клод САМ вирішує наскільки глибоко думати. Спойлер: на деяких запитах він виділяв НУЛЬ токенів на мислення. Просто не думав і галюцинував - вигадував неіснуючі SHA комітів, пакети, версії API
- Знизили дефолтний effort level - по суті урізали якість щоб зекономити на compute

Результат в цифрах:
- Мислення впало на 67%
- Читання файлів перед редагуванням: було 6.6, стало 2.0
- Редагування без читання: з 6% до 34%
- AMD перевели команду на конкурента

Фікс є і він простий. В ~/.claude/settings.json додайте в "env":

"CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING": "1"

Або ще краще - обидва:

"CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING": "1",
"CLAUDE_CODE_EFFORT_LEVEL": "max"

Я поставив собі - клод реально перестав халтурити. Якщо відчуваєте що Claude Code став тупішим за останній місяць - вам не здається, і ось доказ з цифрами.
