**CoALA** (Cognitive Architectures for Language Agents) — академический фреймворк из статьи Princeton/Meta (2023). Описывает как строить память ИИ-агентов по аналогии с человеческой когнитивной архитектурой.

**Ключевые идеи:**

1. **Модульная память:**
  • **Semantic** — факты, знания (наши `facts.md`, `projects/`)
  • **Episodic** — события, опыт (наши дневники `YYYY-MM-DD.md`)
  • **Procedural** — навыки, правила (наш `AGENTS.md`, `SOUL.md`)
2. **Два типа действий:**
  • **Internal** — работа с памятью: retrieval (поиск), reasoning (осмысление), learning (запись)
  • **External** — взаимодействие с миром: API, браузер, файлы
3. **Цикл принятия решений:** perception → retrieval → reasoning → action → learning

**Как это соотносится с нашей системой:**

```| CoALA             | У нас                                   |
| ----------------- | --------------------------------------- |
| Semantic memory   | facts.md, projects/*.md                 |
| Episodic memory   | memory/YYYY-MM-DD.md, weeks/W*.md       |
| Procedural memory | AGENTS.md, SOUL.md, skills/             |
| Retrieval         | memory_search (hybrid embeddings+FTS)   |
| Learning          | запись в дневники + еженедельный дефраг |
```
По сути мы **уже** используем CoALA-подход, не зная что он так называется. Наша структура памяти от 27 февраля — практически 1:1 с тем что описывает фреймворк.
