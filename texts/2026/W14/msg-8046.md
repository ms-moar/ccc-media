# ● Модель не вызывает tool а отвечает текстом. Прокси передаё

**Автор**: sklip
**Дата**: 05.04.2026 09:48
**Топик**: OpenClaw

---

● Модель не вызывает tool а отвечает текстом. Прокси передаёт tools параметр в Claude CLI, но Claude Code CLI игнорирует tools из API запроса — он использует свои встроенные tools (Bash, Edit, Read и т.д.), а не кастомные.

  Вот корень проблемы: Claude Code CLI не поддерживает кастомные tools из запроса. Он знает только свои встроенные. sessions_send — это tool OpenClaw, которого Claude Code CLI не знает.

  OpenClaw при anthropic-messages API ожидает что модель вернёт tool_use block с sessions_send — но прокси через Claude Code CLI никогда его не вернёт.

  Это тупик для мульти-агентной маршрутизации через прокси. Без прямого Anthropic API key или GLM — tools не работают.

  Единственный рабочий вариант без ключей: убрать роутер, main отвечает сам. Бот будет работать, но без мульти-агентности.

  Переключить?
