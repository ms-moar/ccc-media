🧠 Что полезного из утечки Claude Code

1. **Auto-Dream (консолидация памяти)** — можно адаптировать

Claude Code делает "dream" — фоновый процесс который:

• Запускается после 24h И 5+ сессий
• Читает все логи/транскрипты
• Консолидирует память в структурированные файлы
• Удаляет устаревшее, конвертит относительные даты в абсолютные

**Я могу:** запустить аналогичную консолидацию вручную — пройтись по `memory/*.md`, вычистить дубли, обновить устаревшее.

2. **Типы памяти (4 категории)** — уже частично используем

```- user: роль, предпочтения, знания пользователя
- feedback: коррекции поведения (и успехи, не только ошибки!)
- project: ongoing work, дедлайны, кто что делает
- reference: ссылки на внешние системы
```
**Insight:** записывать не только когда меня поправляют, но и когда подход сработал ("yes exactly") — чтобы не дрейфовать от валидированных подходов.

3. **Body structure для feedback-памяти:**

```Rule itself
**Why:** причина (инцидент, предпочтение)  ```**
How to apply: **```когда/где применять
```
