# Проблема У Opus 4.7 новый токенизатор. Тот же текст разбивае

**Автор**: @pionerio
**Дата**: 26.04.2026 02:43
**Топик**: Claude Code

---

Проблема
У Opus 4.7 новый токенизатор. Тот же текст разбивается на +0–35% токенов по сравнению с 4.6. Разброс зависит от контента: проза — нижняя граница, код / JSON / многоязычные тексты / нестандартные символы — верхняя.
Это структурное свойство модели, не баг. Фиксом не снимается.
Что из этого следует:
 1. Контекст кончается раньше — промпт на 150K в 4.6 может стать 180–200K в 4.7.
 2. Счёт растёт при неизменном прайсе — цена за миллион токенов та же, но запрос съедает их на 20–35% больше.
 3. Prompt cache перестаёт попадать — перетокенизированный префикс даёт другой кеш-ключ, хиты падают до прогрева.
 4. Скиллы выходят за лимит body — SKILL.md, укладывавшийся в 4900 токенов, может стать 6200 и попасть в зону деградации (агент начинает пропускать инструкции).
Решение
 1. Кешировать стабильные префиксы — системный промпт, длинные reference-блоки. Один раз заплатить за перетокенизацию, дальше брать из кеша.
 2. Пересчитать лимиты скиллов под 4.7 — прогнать SKILL.md через count_tokens API, ужать всё, что вышло за 5000 токенов.
 3. Реколибровать численные токен-бюджеты — любые лимиты, заданные в цифрах для 4.6, проверить заново.
 4. Замерить реальную дельту на своём контенте — параллельный прогон count_tokens на 4.6 и 4.7 даст точную цифру для конкретного workload, а не диапазон 0–35%.
