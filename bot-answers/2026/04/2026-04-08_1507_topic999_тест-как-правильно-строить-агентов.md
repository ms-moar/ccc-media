## Вопрос

тест: как правильно строить агентов?

## Ответ

**Анализ ситуации с OpenClaw и альтернативы**

По данным из базы знаний группы (февраль-апрель 2026):

**ОСНОВНЫЕ ПРОБЛЕМЫ OPENCLAW:**

1. **Баги в новых версиях** — @Papa_Misha (18.02) предупредил о версии .12 с критическим багом, ломающим всех агентов. @tommypoltev (07.04) столкнулся с бесконечными запросами одобрения на новой версии — решил откатом на 2026.3.13 через NPM.

2. **Потеря контекста при передаче между агентами** — @arbitrageus (23.02) после экспериментов пришёл к выводу: координационный оверхед съедает токены, каждый агент оптимизирует локально. **Оптимально — 2 агента, не 5-6**.

3. **Лимиты контекста** — @Papa_Misha (13.02) предупредил: с >50k токенов начнутся проблемы.

**АЛЬТЕРНАТИВЫ:**

1. Главный агент-менеджер на Opus 4.6 + субагенты на дешёвых моделях
2. ZeroClaw (Rust, 5MB RAM) — для слабых машин
3. Nanobot (Python) — быстрорастущая альтернатива
4. Pi Agents — основа для кастома

**ПАМЯТЬ АГЕНТОВ:**
Структура ~/.openclaw/workspace-{agent}/ с AGENTS.md, MEMORY.md, SOUL.md, USER.md.
